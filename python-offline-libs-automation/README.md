# Python Offline Libs Automation

Single script for downloading, offline installation and removal of Python packages in air-gapped environments.

## Usage

```bash
./offline-libs.sh <command> [options]
```

## Commands

### download
Downloads all packages specified in a `requirements.txt` and generates a compressed package (`offline-packages.tar.gz`).

```bash
./offline-libs.sh download requirements.txt [options]
```

**Options:**
- `--platform`           Target platform (e.g., manylinux2014_x86_64)
- `--python-version`     Python version (e.g., 312, 311)
- `--implementation`     Implementation (default: cp)
- `--abi`                ABI (default: auto)

**Example:**
```bash
./offline-libs.sh download requirements.txt --platform manylinux2014_x86_64 --python-version 312
```

---

### install
Installs Python packages from a package generated by the `download` command.

```bash
./offline-libs.sh install <path/to/package.tar.gz>
```

- Recommended to run inside a virtualenv.

**Example:**
```bash
./offline-libs.sh install offline-packages.tar.gz
```

---

### uninstall
Removes all Python packages installed in the current environment.

```bash
./offline-libs.sh uninstall
```

- Recommended to run inside a virtualenv.
- Script asks for confirmation if not in a virtualenv.

---

## Examples

```bash
# 1. Download dependencies (machine with internet)
./offline-libs.sh download requirements.txt --python-version 312

# 2. Transfer offline-packages.tar.gz to air-gapped environment

# 3. Install (air-gapped environment)
./offline-libs.sh install offline-packages.tar.gz

# 4. If you need to remove
./offline-libs.sh uninstall
```
